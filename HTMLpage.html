Structuring content

header is usually found at the top of a page, contains complany logo or name and main navigation. Can also be used within section to introduce content for headings, subeading, author etc.

footer is usually found at the bottom of a page, contains list of links, copyright information, extra info about comapny, and terms and services.

main wraps around the content of the page, it tells the browser where the main content is.

section wraps around related content. Usually each section has its own heading.

article self contained composition on a web page. It is independenlt distributable or reusable. ex. blog post, product card, interactive widget.

nav its purpose is to provide navigational links to current doument or other documents.

aside used for content that is indirectly related to the main content. Can be found presented on a sidebar for a website.

Accessibility

Landmark roles

header role="banner" role that defines the masthead, added to the primary header.

nav role="navigation" role thats added to the primary navigation at the top.

main role="main" roles that defines an element as the main content of the page.

aside role="complimentary" role that defines an element as being related to prinmary content. Usually added to the aside.

footer role="contentinfo" role that defines element as being information about the content. Like copyright, terms, privacy policy etc. Usually added to the footer.

Labeling links

adding the aria-label="description" is a non visual way of allowing screen readers to read the description

Image Descriptions

adding the alt attribute to images allows it to describe it to someone who cannot view it, and describe it if it doesn't load correctly

Skip Links

Skip links, primarily at the top of the page, allow you to jump to specific areas on a page.

a href="#name" skip link to lead somewhere

main role="main" id="name" place it leads to.
